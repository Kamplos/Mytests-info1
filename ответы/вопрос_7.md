## 1. Веб-страницы и веб-сайты. Разработка интернет-приложений (сайтов)

### 🌐 Веб-страницы и веб-сайты
* **Веб-страница:** Это документ, написанный на языке разметки (HTML), который может содержать текст, изображения, видео и гиперссылки. Страница интерпретируется браузером.
* **Веб-сайт:** Совокупность логически связанных веб-страниц, имеющих общее доменное имя и дизайн.
* **Структура сайта:** 
  * *Главная страница (Home):* Лицо сайта.
  * *Внутренние страницы:* Разделы, статьи, каталоги.
  * *Служебные страницы:* Контакты, политика конфиденциальности.

### 🛠️ Разработка интернет-приложений
Процесс создания сайтов делится на два основных направления:

#### А) Frontend (Клиентская часть)
То, что пользователь видит в браузере.
1. **HTML (HyperText Markup Language):** Каркас страницы, структура контента.
2. **CSS (Cascading Style Sheets):** Внешний вид, стили, цвета, шрифты, адаптивность под мобильные устройства.
3. **JavaScript (JS):** Интерактивность, анимация, динамическое обновление контента без перезагрузки страницы (AJAX).
4. **Фреймворки:** React, Vue.js, Angular — инструменты для ускорения разработки сложных интерфейсов.

#### Б) Backend (Серверная часть)
То, что работает «под капотом» на сервере.
1. **Языки программирования:** Python (Django, Flask), PHP (Laravel), JavaScript (Node.js), Java, C#.
2. **Базы данных (DBMS):** Хранение информации о пользователях, товарах, постах (MySQL, PostgreSQL, MongoDB).
3. **API (Application Programming Interface):** Интерфейс взаимодействия между фронтендом и бэкендом (часто в формате JSON).

#### В) Полный цикл разработки (Fullstack)
Включает также проектирование UI/UX (дизайн интерфейса), тестирование, деплой (размещение на хостинге) и поддержку.

## 2. Задача на объем информации и кодировку текста

### 📋 Условие
* Исходная кодировка: **Unicode** (всего 65 536 символов).
* Конечная кодировка: **Windows-1251** (всего 256 символов).
* Изменение объема: сообщение уменьшилось на **3912 бит**.
* Найти: длину сообщения в символах ($K$).

### 🧮 Решение

**Шаг 1. Определяем вес одного символа в каждой кодировке**
Количество бит, необходимых для кодирования одного символа, зависит от мощности алфавита ($N$) по формуле $2^i = N$.

1. **Для Unicode:**
   $$N_1 = 65\,536 = 2^{16}$$
   Значит, на один символ уходит **16 бит** ($i_1 = 16$).

2. **Для Windows-1251:**
   $$N_2 = 256 = 2^8$$
   Значит, на один символ уходит **8 бит** ($i_2 = 8$).

**Шаг 2. Составляем уравнение**
Пусть $K$ — количество символов в сообщении.
* Исходный объем сообщения: $V_1 = K \times 16$ бит.
* Конечный объем сообщения: $V_2 = K \times 8$ бит.

По условию, разница между объемами составляет 3912 бит:
$$V_1 - V_2 = 3912$$
$$16K - 8K = 3912$$

**Шаг 3. Решаем уравнение**
$$8K = 3912$$
$$K = \frac{3912}{8}$$

Выполним деление:
$$3912 : 8 = 489$$

**Шаг 4. Проверка**
* Было: $489 \times 16 = 7824$ бит.
* Стало: $489 \times 8 = 3912$ бит.
* Разница: $7824 - 3912 = 3912$ бит. Всё верно.

### ✅ Ответ
> **Длина сообщения составляет 489 символов.**
