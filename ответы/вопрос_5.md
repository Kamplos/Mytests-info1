## 1. Прикладное ПО. Специализированное прикладное ПО. САПР

### 📊 Прикладное программное обеспечение (Application Software)
Прикладное ПО предназначено для выполнения конкретных пользовательских задач, не связанных с управлением самим компьютером.
* **Офисные пакеты:** Обработка текстов, таблиц и презентаций (Microsoft Office, LibreOffice).
* **Мультимедиа:** Просмотр видео, прослушивание музыки, обработка фотографий и видео.
* **Коммуникационное ПО:** Браузеры, почтовые клиенты, мессенджеры.
* **Образовательное и развлекательное ПО:** Электронные учебники, тренажеры, компьютерные игры.

### 🛠️ Специализированное прикладное ПО
Это программы, разработанные для решения узкопрофессиональных задач в特定нных отраслях.
* **Бухгалтерские системы:** 1С:Предприятие, SAP (автоматизация учета, расчета зарплат, налоговой отчетности).
* **Медицинское ПО:** Системы хранения медицинских изображений (PACS), электронные медицинские карты.
* **Банковское ПО:** Системы клиент-банк, процессинговые центры для обработки транзакций.
* **Геоинформационные системы (ГИС):** Работа с картами и пространственными данными (ArcGIS, QGIS).

### 🏗️ САПР (CAD — Computer-Aided Design)
**Система автоматизированного проектирования** — это программный комплекс, используемый инженерами, архитекторами и конструкторами для создания чертежей, 3D-моделей и технической документации.
* **Основные функции:** 
  * Двумерное черчение (2D) и трехмерное моделирование (3D).
  * Инженерные расчеты и симуляция физических процессов (CAE).
  * Подготовка данных для станков с ЧПУ (CAM).
* **Популярные системы:** AutoCAD, Компас-3D, SolidWorks, Revit, NanoCAD.
* **Значение:** САПР позволяет значительно сократить время разработки, повысить точность расчетов и визуализировать объект до начала его физического производства.

## 2. Задача на системы счисления

### 📋 Условие
Найти **наименьшее четырёхзначное восьмеричное число**, двоичная запись которого содержит ровно **5 единиц**.

### 🧮 Решение

**Шаг 1. Анализ структуры числа**
* Число должно быть **четырёхзначным в восьмеричной системе**. Наименьшее такое число — $1000_8$.
* В восьмеричной системе одна цифра заменяет ровно **3 бита** в двоичной записи ($8 = 2^3$).
* Значит, четырёхзначное восьмеричное число в двоичной системе будет занимать от $9$ до $12$ бит.

**Шаг 2. Поиск минимального значения**
Чтобы число было наименьшим, старшие разряды должны быть равны нулю (или минимально возможному значению), а единицы должны стоять в младших разрядах.

Восьмеричное число имеет вид: $d_3 d_2 d_1 d_0$.
Переведем каждую восьмеричную цифру в тройку бит:
* $0_8 \rightarrow 000_2$
* $1_8 \rightarrow 001_2$
* ...
* $7_8 \rightarrow 111_2$

Нам нужно получить ровно **5 единиц** во всей двоичной записи. Чтобы число было минимальным, будем заполнять единицами самые младшие позиции справа налево.

Попробуем собрать число с конца:
1. Младшая восьмеричная цифра ($d_0$): может дать максимум 3 единицы (`111` = $7_8$). Осталось разместить $5 - 3 = 2$ единицы.
2. Следующая цифра ($d_1$): должна содержать оставшиеся 2 единицы. Чтобы число было минимальным, эти единицы должны быть в младших битах этой тройки: `011` = $3_8$.
3. Старшие цифры ($d_2$ и $d_3$): должны быть минимальными. Так как мы уже разместили все 5 единиц, остальные биты должны быть нулями.
   * $d_2 = 000_2 = 0_8$
   * $d_3 = 001_2 = 1_8$ (так как число должно быть четырёхзначным, старший разряд не может быть 0).

**Проверка полученного числа $1037_8$:**
* $1_8 \rightarrow 001$
* $0_8 \rightarrow 000$
* $3_8 \rightarrow 011$
* $7_8 \rightarrow 111$
* Двоичная запись: $001\,000\,011\,111_2$.
* Количество единиц: $1 + 0 + 2 + 3 = 6$. **Ошибка!** Мы получили 6 единиц. Нам нужно ровно 5.

**Корректировка:**
Нам нужно убрать одну единицу из старших доступных позиций, чтобы сохранить минимальность числа, но уменьшить их количество.
Вернемся к распределению 5 единиц по тройкам бит (справа налево):
* Тройка 0 (младшая): `111` (3 единицы) $\rightarrow 7_8$. Осталось 2 единицы.
* Тройка 1: `011` (2 единицы) $\rightarrow 3_8$. Осталось 0 единиц.
* Тройка 2: `000` (0 единиц) $\rightarrow 0_8$.
* Тройка 3 (старшая): Должна быть ненулевой, так как число 4-значное. Минимальная ненулевая восьмеричная цифра — $1$ (`001`). Но это добавит еще 1 единицу. Итого: $3+2+0+1=6$. Опять 6.

Значит, конфигурация `1037` дает 6 единиц. Нам нужно где-то "потерять" одну единицу, сместив её в более старший разряд или изменив структуру.

Давайте попробуем другое распределение 5 единиц:
* Чтобы число было минимальным, старший разряд $d_3$ должен быть как можно меньше. Пусть $d_3 = 1$ (`001`, 1 единица).
* Осталось разместить $5 - 1 = 4$ единицы в младших трех разрядах ($d_2, d_1, d_0$).
* Чтобы число было минимальным, $d_2$ должен быть 0 (`000`, 0 единиц).
* Осталось разместить 4 единицы в $d_1$ и $d_0$.
* Максимум в одной восьмеричной цифре — 3 единицы.
* Пусть $d_0 = 7$ (`111`, 3 единицы). Осталась 1 единица для $d_1$.
* $d_1$ с одной единицей в младшем положении: `001` = $1_8$.

Получаем число: **$1017_8$**.

**Проверка $1017_8$:**
* $1 \rightarrow 001$
* $0 \rightarrow 000$
* $1 \rightarrow 001$
* $7 \rightarrow 111$
* Двоичная запись: $001\,000\,001\,111_2$.
* Считаем единицы: $1 + 0 + 1 + 3 = 5$. **Верно!**

Сравним с предыдущим вариантом $1037_8$ (который не подошел по количеству) и проверим, нет ли числа меньше $1017_8$ с 5 единицами.
Числа вида $100X_8$:
* $1007_8 \rightarrow 001\,000\,000\,111$ (4 единицы) — мало.
* $1017_8$ — подходит (5 единиц).

Числа вида $10XX_8$, где вторая цифра больше 0, будут больше $1017_8$.
Числа вида $11XX_8$ будут еще больше.

Таким образом, наименьшее число — $1017_8$.

### ✅ Ответ
> **1017**
