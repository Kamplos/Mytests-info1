## 1. Принципы построения и аппаратные компоненты компьютерных сетей. Сетевые протоколы. Локальная сеть

### 🌐 Принципы построения сетей
Компьютерная сеть — это система связи двух и более компьютеров для обмена данными и совместного использования ресурсов.
* **Топологии:** Звезда (наиболее популярная), Шина, Кольцо, Ячеистая (Mesh).
* **Масштаб:** PAN (персональная), LAN (локальная), MAN (городская), WAN (глобальная/Интернет).
* **Клиент-серверная архитектура:** Выделенный сервер предоставляет услуги, клиенты их потребляют.
* **Одноранговая сеть (P2P):** Все узлы равноправны.

### 💻 Аппаратные компоненты
1. **Сетевая карта (NIC):** Преобразует данные компьютера в сигналы для передачи по среде. Имеет уникальный MAC-адрес.
2. **Коммутатор (Switch):** Работает на канальном уровне (L2). Пересылает кадры только тому порту, где находится получатель (по MAC-адресу).
3. **Маршрутизатор (Router):** Работает на сетевом уровне (L3). Соединяет разные сети (например, домашнюю локальную сеть и Интернет), выбирает оптимальный путь для пакетов данных (по IP-адресам).
4. **Точки доступа (Wi-Fi):** Обеспечивают беспроводное подключение устройств к сети.
5. **Кабели и разъемы:** Витая пара (UTP) с разъемами RJ-45, оптоволокно.

### 📡 Сетевые протоколы
Протокол — это набор правил обмена данными. Обычно используется стек протоколов TCP/IP:
* **IP (Internet Protocol):** Отвечает за адресацию (IP-адреса) и маршрутизацию пакетов.
* **TCP (Transmission Control Protocol):** Гарантирует надежную доставку данных (с подтверждением приема).
* **UDP (User Datagram Protocol):** Быстрая доставка без гарантий (используется в онлайн-играх, видеостриминге).
* **HTTP/HTTPS:** Протоколы для передачи веб-страниц.
* **DNS:** Преобразует доменные имена (yandex.ru) в IP-адреса.

### 🏠 Локальная вычислительная сеть (ЛВС / LAN)
* **Назначение:** Объединение компьютеров в пределах одного здания или офиса для совместного доступа к принтерам, файлам и базам данных.
* **Адресация:** Используются частные (внутренние) IP-адреса (диапазоны 192.168.x.x, 10.x.x.x).
* **Безопасность:** Ограничение доступа через пароли, фаерволы и VLAN (виртуальные локальные сети).

## 2. Задача на системы счисления

### 📋 Условие
* Число $38_{10}$ в системе с основанием $N$ оканчивается на **2**.
* Число $59_{10}$ в системе с основанием $N$ оканчивается на **5**.
* Найти: наименьшее возможное основание $N$.

### 🧮 Решение

**Шаг 1. Анализ условия окончания числа**
Если запись числа $A$ в системе с основанием $N$ оканчивается на цифру $d$, то это означает, что остаток от деления $A$ на $N$ равен $d$.
Математически это записывается как:
$$A \equiv d \pmod N$$
или
$$A - d \text{ делится на } N \text{ без остатка.}$$

Также важно помнить ограничение: основание системы счисления $N$ должно быть строго больше любой цифры, используемой в записи.
* Так как есть цифра 5, то **$N > 5$**, то есть $N \ge 6$.

**Шаг 2. Составляем уравнения**
1. Для числа 38:
   $$38 - 2 = 36 \text{ делится на } N$$
   Значит, $N$ является делителем числа 36.

2. Для числа 59:
   $$59 - 5 = 54 \text{ делится на } N$$
   Значит, $N$ является делителем числа 54.

**Шаг 3. Находим общие делители**
Нам нужно найти такие $N$, которые являются делителями и 36, и 54. То есть $N$ должно быть общим делителем этих чисел.
Найдем НОД (Наибольший Общий Делитель) или просто перечислим делители.

* Делители числа 36: 1, 2, 3, 4, 6, 9, 12, 18, 36.
* Делители числа 54: 1, 2, 3, 6, 9, 18, 27, 54.

Общие делители: **1, 2, 3, 6, 9, 18**.

**Шаг 4. Применяем ограничение $N \ge 6$**
Из списка общих делителей $\{1, 2, 3, 6, 9, 18\}$ выбираем те, что больше или равны 6.
Подходящие значения: $6, 9, 18$.

**Шаг 5. Выбираем наименьшее**
Наименьшее из подходящих значений — **6**.

*Проверка для $N=6$:*
* $38 : 6 = 6$ (остаток 2). Запись заканчивается на 2. Верно.
* $59 : 6 = 9$ (остаток 5). Запись заканчивается на 5. Верно.

### ✅ Ответ
> **6**
