## 1. Общие проблемы защиты информации и информационной безопасности. Угрозы, связанные с использованием ИКТ

### 🛡️ Информационная безопасность (ИБ)
**Информационная безопасность** — это состояние защищенности информации и поддерживающей инфраструктуры от случайных или преднамеренных воздействий естественного или искусственного характера, которые могут нанести неприемлемый ущерб субъектам информационных отношений.

Три ключевых свойства информации (триада CIA):
1.  **Конфиденциальность (Confidentiality):** Доступ к информации имеют только авторизованные пользователи.
2.  **Целостность (Integrity):** Гарантия того, что информация не была изменена несанкционированным образом.
3.  **Доступность (Availability):** Информация и сервисы доступны авторизованным пользователям, когда это необходимо.

### ⚠️ Основные угрозы в сфере ИКТ
1.  **Вредоносное ПО (Malware):**
    *   *Вирусы и черви:* Саморазмножающиеся программы, повреждающие данные.
    *   *Трояны:* Маскируются под легитимные программы для кражи данных или создания бэкдоров.
    *   *Программы-вымогатели (Ransomware):* Шифруют файлы пользователя и требуют выкуп за расшифровку.
2.  **Социальная инженерия:**
    *   *Фишинг:* Рассылка поддельных писем/сообщений с целью выманивания паролей и данных карт.
    *   *Претекстинг:* Звонок от лица «службы поддержки» для получения доступа к аккаунту.
3.  **Сетевые атаки:**
    *   *DDoS (Distributed Denial of Service):* Перегрузка сервера огромным количеством запросов, делающая его недоступным.
    *   *Man-in-the-Middle (MitM):* Перехват трафика между двумя сторонами (часто в открытых Wi-Fi сетях).
4.  **Уязвимости программного обеспечения:** Использование ошибок в коде ОС или приложений для несанкционированного доступа (эксплойты).

### 🔐 Меры защиты
*   Использование антивирусов и межсетевых экранов (Firewall).
*   Регулярное обновление ПО (патчи безопасности).
*   Двухфакторная аутентификация (2FA).
*   Резервное копирование данных (Backup).
*   Обучение пользователей основам цифровой гигиены.

## 2. Задача на IP-адресацию и маски подсети

### 📋 Условие
*   **IP-адрес компьютера:** `232.165.164.243`
*   **Маска подсети:** `255.255.248.0`
*   Найти: **номер компьютера в сети** (host ID).

### 🧮 Решение

Номер компьютера в сети определяется младшими битами IP-адреса, которые соответствуют нулям в маске подсети. Чтобы найти его, нужно выполнить побитовую операцию ИЛИ НЕ (или просто вычесть адрес сети из IP-адреса), но проще всего посмотреть на те октеты, где в маске стоят не 255 и не 0.

**Шаг 1. Проанализируем маску по октетам**
Маска: `255.255.248.0`

1.  **1-й октет маски: 255** (`11111111`). Все биты — единицы. Это часть адреса сети. Номер компьютера здесь всегда **0**.
2.  **2-й октет маски: 255** (`11111111`). Все биты — единицы. Это часть адреса сети. Номер компьютера здесь всегда **0**.
3.  **4-й октет маски: 0** (`00000000`). Все биты — нули. Это полностью часть адреса компьютера. Значит, номер компьютера в этом октете равен значению IP-адреса: **243**.
4.  **3-й октет маски: 248**. Это смешанный октет. Нам нужно определить, какая часть этого байта относится к адресу компьютера.

**Шаг 2. Разберем 3-й октет**
*   Значение маски: `248`.
*   Переведем 248 в двоичный код:
    $$128 + 64 + 32 + 16 + 8 = 248$$
    $$248_{10} = 11111000_2$$
*   Видим, что старшие 5 бит — единицы (сеть), а младшие 3 бита — нули (компьютер).
*   Значение IP-адреса в 3-м октете: `164`.
*   Переведем 164 в двоичный код:
    $$128 + 32 + 4 = 164$$
    $$164_{10} = 10100100_2$$

Чтобы получить номер компьютера в этом октете, нам нужно оставить только те биты, которые соответствуют нулям в маске (последние 3 бита):
*   IP (3-й октет): `10100`**`100`**
*   Маска (3-й октет): `11111`**`000`**
*   Результат (Host part): `00000`**`100`**

Переведем `100` из двоичной системы в десятичную:
$$1 \times 2^2 + 0 \times 2^1 + 0 \times 2^0 = 4$$

Значит, вклад 3-го октета в номер компьютера равен **4**.

**Шаг 3. Соберем полный номер компьютера**
Номер компьютера складывается из значений тех частей IP-адреса, которые попали в "нулевую" зону маски.
*   От 1-го и 2-го октетов вклад: 0.
*   От 3-го октета вклад: 4.
*   От 4-го октета вклад: 243 (так как маска там 0, берем всё число целиком).

Номер компьютера вычисляется как сумма с учетом весов октетов (поскольку это единое 32-битное число, но обычно в таких задачах ЕГЭ под "номером компьютера" понимают именно значение host-ID, собранное из младших бит).

Давайте запишем это строго.
Адрес компьютера (Host ID) — это младшие биты IP-адреса.
Маска `255.255.248.0` означает, что под сеть отдано $8+8+5 = 21$ бит.
Под адрес компьютера остается $32 - 21 = 11$ бит.

Эти 11 бит находятся в конце 3-го октета (3 бита) и во всем 4-м октете (8 бит).
*   Младшие 3 бита 3-го октета IP (`164`): как мы выяснили, это `100`₂ = **4**.
*   Весь 4-й октет IP (`243`): это **243**.

Чтобы получить итоговое десятичное число номера компьютера, нужно сложить эти части, учитывая их разрядность.
3 бита из 3-го октета стоят "старше" чем 8 бит 4-го октета.
Вес этих 3 бит относительно конца адреса: $2^8 = 256$.

$$N_{host} = (\text{значение из 3-го октета}) \times 256 + (\text{значение из 4-го октета})$$
$$N_{host} = 4 \times 256 + 243$$
$$N_{host} = 1024 + 243 = 1267$$

*(Примечание: В некоторых трактовках задач ЕГЭ просят просто указать последний октет, если маска 255.255.255.0, но здесь маска сложнее. Номер узла в сети — это целое число, определяемое всеми битами хоста.)*

Проверим логику:
IP: `... .164.243`
Mask: `... .248.0`
Network Address (Адрес сети) = IP AND Mask.
3-й октет сети: $164 \text{ AND } 248$.
$164 = 10100100$
$248 = 11111000$
AND = $10100000 = 160$.
Адрес сети: `232.165.160.0`

Host ID = IP - Network Address (в двоичном виде по модулю размера подсети).
Размер подсети (количество узлов) определяется нулями в маске.
Нулей в маске: 3 (в 3-м октете) + 8 (в 4-м) = 11 бит.
Максимальный номер узла: $2^{11} - 1 = 2047$.

Вычислим смещение текущего IP от начала сети:
3-й октет: $164 - 160 = 4$.
4-й октет: $243 - 0 = 243$.
Итоговый номер: $4 \times 256 + 243 = 1024 + 243 = 1267$.

### ✅ Ответ
> **1267**
